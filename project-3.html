🚀 Development Journey & Self-Assessment

📋 Project Architecture Planning

Started by analyzing the Noroff API structure and mapping out component hierarchy. Created a detailed plan for state management, routing structure, and component reusability before writing any code.

🎯 Key Improvement: Component Structure

Original approach: Initially created large, monolithic components handling multiple responsibilities.

Improvement made: Refactored into smaller, reusable components following single responsibility principle, improving code maintainability and testing capabilities.

⚙️ React Development & State Management

Implemented modern React patterns using functional components and hooks. Created custom hooks for API calls and used Context API for global cart state management across components.

🎯 Key Improvement: Search Performance

Challenge faced: Initial search implementation caused performance issues with real-time filtering on large product lists.

Solution implemented: Added debouncing to search input and optimized filtering logic, reducing API calls by 80% and improving user experience.

🔧 Code Quality Improvements Based on Review Feedback

After conducting a thorough code review, I identified and fixed several code quality issues to improve maintainability and professional standards:

🎯 Code Quality Enhancement: Professional Standards

Issues Identified: Inconsistent indentation, trailing whitespace, console.log statements, and unused variables affecting code quality.

Improvements Made: Standardized code formatting, removed debugging statements, cleaned up unused imports, and implemented consistent styling throughout the codebase.

  • Consistent Indentation: Fixed inconsistent code indentation in src/api/index.ts, standardizing to 2-space indentation throughout the project
  • Removed Trailing Whitespace: Cleaned up trailing spaces in src/components/ProductCard.tsx and other files for cleaner git diffs
  • Production-Ready Code: Removed console.log statements from src/api/index.ts to eliminate debugging code from final submission
  • Code Cleanup: Removed unused variables like 'err' in src/pages/HomePage.tsx to reduce code complexity
  • Enhanced Documentation: Updated README.md file with improved project documentation and setup instructions

💡 Technical Challenges Overcome

🔄 State Management Complexity

Challenge: Managing cart state across multiple components and persisting data between sessions.

Solution: Implemented React Context for global state management combined with localStorage for persistence, ensuring seamless user experience across page refreshes.

💰 Dynamic Pricing Calculations

Challenge: Calculating and displaying discounts dynamically when price differences exist between regular and discounted prices.

Solution: Created utility functions to handle price calculations and discount percentages, with proper formatting for currency display and clear visual indicators for sales.

🔍 Search Functionality Optimization

Challenge: Implementing efficient real-time search that works smoothly with large product datasets.

Solution: Implemented debouncing techniques and optimized filtering algorithms to provide instant search results without performance degradation.

📱 Responsive Layout Challenges

Challenge: Creating a shopping experience that works seamlessly across all device sizes and orientations.

Solution: Used CSS Grid and Flexbox with mobile-first approach, implementing responsive breakpoints and touch-friendly interface elements.
